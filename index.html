01
AI Workspace · Frontend Lead

Multi-Model AI Workspace

An Angular workspace where a single query fans out to 9–10 AI models at once — Claude, GPT, Gemini and more, side by side — so users compare answers in one glance instead of hopping between tools.

Claude OpenAI Gemini +6–7 more
AngularTypeScriptRxJSStreaming UIWeb Speech
Problem

Comparing AI models meant juggling several tabs, re-typing the same prompt into each, and stitching answers together by hand. There was no single place to ask once and see how every model responded.

Frontend approach

I led the Angular frontend: a workspace that broadcasts one query to 9–10 models simultaneously and renders each reply as a live, streaming chat panel side by side — plus hands-free dictation and transcription with multi-language support, wired cleanly into the backend AI services.

Impact
  • Ask once, compare every model's answer in one view
  • Parallel streaming chats replaced tab-hopping
  • Hands-free input via dictation & transcription
  • Multi-language support widened who could use it

Problem

The hard part wasn't generating the code — it was making the UI feel instant while orchestrating multi-step LLM generation, schema synthesis and cloud deploy under the hood. Founders without engineering teams were burning quarters of runway just to validate a hypothesis.

Frontend approach

I led the frontend architecture: a streaming-first shell where every generation step renders progressively, with optimistic step previews and a full-screen diff viewer that handles five-thousand-line outputs without dropping a frame. State coordinated through a deliberate split — NgRx for the run lifecycle, Signals for everything ephemeral.

The point isn't the framework. The point is that a non-technical founder ships a working prototype before lunch.

04
AI Search · Climate Tech

AI Document Search Engine

A semantic document search and retrieval experience for a climate-tech nonprofit — turning a sprawling corpus of policy, program and incentive docs into instant answers grounded in their source passages.

Semantic SearchVector EmbeddingsRAG PipelineRetrieval API
Problem

The client's research and program documents had grown into a large, unstructured corpus. Stakeholders spent hours digging through PDFs for clauses, rebates and program details — and keyword search fell short, since the same idea gets phrased a dozen ways across revisions.

Frontend approach

I helped deliver the search experience behind their existing product UI — ingestion and chunking, vector embeddings, a semantically indexed retrieval layer, and a clean API returning ranked passages with grounded citations, tuned for the shape of policy and incentive content.

Impact
  • Hours of manual hunting compressed into seconds of search
  • Citation-backed answers link straight to the source passage
  • Indexing ingests a growing corpus without manual curation
  • Drop-in API integrated cleanly with the existing product
05
AI Platform · Developer Tools

Trillo Backend Workbench

An IDE-style backend workbench for Trillo.ai where developers compose, test and deploy AI-powered workflows. The same foundation later powered the Trillo File Manager product.

Angular 18SignalsNgRxTypeScriptVirtualization
Problem

Trillo's customers were constantly context-switching between editors, runtime logs, AI prompt tools and admin consoles. The product needed one workspace that felt as responsive as a desktop IDE — without ballooning the bundle or fighting change detection at scale.

Frontend approach

I helped architect the workbench around lazy-loaded feature modules, Signals + OnPush for surgical re-renders, and a virtualized log/console layer that holds tens of thousands of entries without dropping frames. Reusable layout primitives extended cleanly into the File Manager.

Impact
  • IDE-style UX — split panes, live logs, AI tooling — without lag
  • New feature surfaces shipped in days, not weeks
  • Became a core surface for Trillo's developer customers
  • Architecture reused for the Trillo File Manager spin-off
