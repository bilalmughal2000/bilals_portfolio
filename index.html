01
AI Workspace · Frontend Lead

Multi-Model AI Workspace

An Angular workspace where a single query fans out to 9–10 AI models at once — Claude, GPT, Gemini and more, side by side — so users compare answers in one glance instead of hopping between tools.

Claude OpenAI Gemini +6–7 more
AngularTypeScriptRxJSStreaming UIWeb Speech
Problem

Comparing AI models meant juggling several tabs, re-typing the same prompt into each, and stitching answers together by hand. There was no single place to ask once and see how every model responded.

Frontend approach

I led the Angular frontend: a workspace that broadcasts one query to 9–10 models simultaneously and renders each reply as a live, streaming chat panel side by side — plus hands-free dictation and transcription with multi-language support, wired cleanly into the backend AI services.

Impact
  • Ask once, compare every model's answer in one view
  • Parallel streaming chats replaced tab-hopping
  • Hands-free input via dictation & transcription
  • Multi-language support widened who could use it
02
AI Platform · Enterprise · Google Cloud Premier Partner

Trillo AI Platform

Frontend for Trillo AI — an application platform that turns plain-English requirements into running, enterprise-class apps: auth, database, APIs, workflows and agents. Deploy on Trillo AOS in one click, or generate full-stack code you own.

AngularTypeScriptTailwind CSSRESTful APIsResponsive UI
Problem

Work that traditionally takes 6–12 months needed to ship in days — without CI/CD, DevOps or cloud complexity. Both technical and non-technical users had to move from a plain-English description → blueprint → deployed app, all through one coherent product surface.

Frontend approach

I built responsive Angular & TypeScript interfaces across the platform's describe → blueprint & code → deploy flow — the requirement input, generated spec and blueprint views, and the two build paths (managed deploy on Trillo AOS vs. full-stack code generation). Integrated platform APIs and kept it fast with lazy loading and code splitting.

Impact
  • Part of a platform trusted by 1,000+ teams, 100+ customers
  • Enterprise-class from day one — Google Cloud Premier Partner
  • Plain-English → deployed backend (auth, DB, APIs, agents) in minutes
  • One surface for both managed AOS deploys and full code ownership
03
AI Search · Climate Tech

AI Document Search Engine

A semantic document search and retrieval experience for a climate-tech nonprofit — turning a sprawling corpus of policy, program and incentive docs into instant answers grounded in their source passages.

Semantic SearchVector EmbeddingsRAG PipelineRetrieval API
Problem

The client's research and program documents had grown into a large, unstructured corpus. Stakeholders spent hours digging through PDFs for clauses, rebates and program details — and keyword search fell short, since the same idea gets phrased a dozen ways across revisions.

Frontend approach

I helped deliver the search experience behind their existing product UI — ingestion and chunking, vector embeddings, a semantically indexed retrieval layer, and a clean API returning ranked passages with grounded citations, tuned for the shape of policy and incentive content.

Impact
  • Hours of manual hunting compressed into seconds of search
  • Citation-backed answers link straight to the source passage
  • Indexing ingests a growing corpus without manual curation
  • Drop-in API integrated cleanly with the existing product
04
Analytics · Enterprise

Data Visualization Platform

Migration of 60+ Looker Studio charts into a fully owned, high-performance Angular analytics product.

Angular 18SignalsChart.jsApache EChartsTypeScript
Problem

The business leaned on 15+ Looker Studio reports that were slow to load, costly to license, and impossible to extend. Stakeholders needed deep customization, embeddable views and tighter integration with internal data.

Frontend approach

I built a modular charting layer wrapping Chart.js and Apache ECharts behind one unified Angular API — lazy-loaded report modules, virtualized lists for large datasets, and OnPush + Signals to keep change detection cheap. Reusable primitives dropped new chart types in days.

Impact
  • 60+ charts across 15+ reports migrated with full parity
  • Significant load-time and interactivity gains
  • Eliminated the recurring Looker Studio licensing cost
  • Unified API let new report types ship in days, not weeks
05
AI Platform · Developer Tools

Trillo Backend Workbench

An IDE-style backend workbench for Trillo.ai where developers compose, test and deploy AI-powered workflows. The same foundation later powered the Trillo File Manager product.

Angular 18SignalsNgRxTypeScriptVirtualization
Problem

Trillo's customers were constantly context-switching between editors, runtime logs, AI prompt tools and admin consoles. The product needed one workspace that felt as responsive as a desktop IDE — without ballooning the bundle or fighting change detection at scale.

Frontend approach

I helped architect the workbench around lazy-loaded feature modules, Signals + OnPush for surgical re-renders, and a virtualized log/console layer that holds tens of thousands of entries without dropping frames. Reusable layout primitives extended cleanly into the File Manager.

Impact
  • IDE-style UX — split panes, live logs, AI tooling — without lag
  • New feature surfaces shipped in days, not weeks
  • Became a core surface for Trillo's developer customers
  • Architecture reused for the Trillo File Manager spin-off
06
Project Management · AI-Built

Molecule Hierarchy PMS

A complex hierarchical project-management platform modeled on molecule-style decomposition, where every node can branch into deeply nested sub-projects.

Next.jsReactTypeScriptTailwindFirebase
Problem

Teams running multi-layered programs were juggling spreadsheets and flat task lists that hid critical relationships. They needed to see how parent goals decomposed into child workstreams — and back — without losing context across hundreds of nodes.

Frontend approach

Built end-to-end with an AI pair-programming workflow: a Next.js + TypeScript app with a virtualized hierarchy renderer, drag-and-drop reorganization, role-based permissions and real-time collaboration. The architecture stayed human-driven while AI accelerated iteration.

Impact
  • Production launch in a fraction of the typical timeline
  • Hundreds of nodes rendered smoothly via virtualization
  • Drag-and-drop reorg with optimistic updates & conflict resolution
  • Replaced spreadsheet workflows with a single source of truth
